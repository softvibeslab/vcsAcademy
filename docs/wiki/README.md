## Documentation Standards

### Formatting
- Use Markdown for all documentation
- Include code examples with proper syntax highlighting
- Add diagrams where helpful (using Mermaid or ASCII art)
- Keep sections focused and well-organized

### Structure
- Start with a brief overview
- Use clear headings and subheadings
- Include table of contents for long pages
- Add navigation links at the bottom

### Maintenance
- Keep documentation up to date with code changes
- Review and update monthly
- Remove outdated information
- Add examples for new features

## Contributing to Documentation

We welcome documentation improvements! To contribute:

1. Edit the relevant markdown file
2. Follow the existing formatting style
3. Test any code examples
4. Submit a pull request with "docs:" prefix
